Abstract

 

 

The document processing application is a system for the extraction analysis and classification of documents into various predefined categories. The purpose of the system is to provide an online source for businesses to process documents by utilising deep learning and natural language techniques to extract analyse and categorize documents which will help to enhance business processes within an information system.

 

The project would assist in streamlineing the initial investigation process into document discovery by performing an initial triage of the documents to be examined. This would be achieved by using optical character recognition to read in the files and create a dataset that can be used with deep learning models to extract topical information and to classify the files by topic into various categories that can be easily scanned by the user. The system will also be able to provide a sentiment analyses on documents.

 

The web application will be using a three-tier architecture that is based on the client-server model.

The project is a web-based system developed in the Django framework, also including front-end technologies such as JavaScript, Bootstrap, HTML, and CSS. It uses an SQL database for storage and queries. This will be implemented using amazon web services for the hosting of the ec2 server.

Introduction

 

This project is designed to address a business process shortfall of companies within the small to medium enterprise (S.M.E.) sector, to access a document processing system that is capable of dealing with document sets without the need to hire specialists in the area.

Project objectives

This project hopes to address this gap within the business community by creating a distributed application in which a user can upload a set of documents to a web application and perform the following tasks:

1) extract textual content from a document using optical character recognition (O.C.R.).

2) analyse the extracted text and perform topic extraction.

3) Utilize the extracted topic for classification purposes.

4)perform sentiment analysis on a document.

5) create a searchable index for the documents.

Machine Learning.

Machine learning is programming computers to optimize a performance criterion using example data or past experience. (Alpaydin, 2014, p. 27) when human expertise does not exist or when humans are unable to explain their expertise.

There are four major categories of machine learning:

       Inductive based learning : this is where the system tries to induce a general rule from a set of observed instances or examples.

       Similarity-based learning:- This is used to estimate the class label of a test sample based on the similarities between the test sample and a set of labelled training samples, and the pairwise similarities between the training samples

       Probability-based learning

       Error based learning

Bag of Words

One of the simplest algorithms for category recognition is the Bag-of-Words (B.O.W.) model. The B.O.W. approach attempts to represent objects, and images as unordered collections of feature descriptors The Bag of Words approach simply computes the distribution of visual words (features) found in the query document and compares this distribution to those found in the training set. In the B.O.W. model, the source can be treated as a document, and then the algorithm tries to search for words or features in the image to define this document.

Building Predictive Models

 

Building predictive models is an iterative process in which a model is created from an initial hypothesis and then refined until it produces a valuable business outcome. Statistical analysis, data mining, or data visualization tools may be needed to run a predictive model.

These six steps will help you develop and use predictive models.

1. Scope and define the predictive analytics model you want to build. In this step, you want to determine what processes will be analysed and what the desired business outcomes area, such as the adoption of a product by a certain segment of customers.

2. Explore and profile your data. Predictive analytics is data intensive. In this step, you need to determine the data that is needed, where it is stored and whether it is readily accessible, and its current state.

3. Gather, cleanse, and integrate the data. Once you know where the necessary data is located, you may need to clean the data. You will want to build your model from a consistent and comprehensive set of information that is ready to be analysed.

4. Build the predictive model. Establish the hypothesis and then build the test model. Your goal is to include and rule out different variables and factors and test the model using historical data to see if the results produced by the model to prove the hypothesis.

5. Incorporate analytics into business processes. To make the model of value, you need to integrate it into the business process so it can be used to help achieve the outcome.

6. Monitor the model and measure the business results. We live in a dynamic environment where buying, competitive, and other factors change. You need to monitor the model and measure how effective it is at continuing to produce the desired outcome. It may be necessary to adjust and fine-tune the model as conditions evolve.

 

 

Classification

Classification is a data mining technique or function that assigns items from a data set to a target category or class. The goal of classification is to accurately predict the target classes for each case in the data. The purpose of classification within the future development of this project will be used to accurately classify documents from the dataset. By analysing the text within the document, we can then classify the document accordingly. The aim of this process is to reduce the manual hours that would be needed to accomplish this task.

A classification task starts with a training data set in which class assignments are known for each case. These class assessments are the values of a particular target. There are different types of classes. These classes are distinct and are not presented in an ordered relationship to each other. Non ordered values normally indicate a categorical target rather than a numerical target value. It is important to note that a predictive model with a categorical target uses classification algorithms while predictive models whose target values are numerical use regression algorithms.

A simple understanding of how classification algorithms work for the purpose of our classification model: upon completion of the O.C.R., the model will be used to apply a bag of words approach to the data set. The model will then apply pre-existing algorithms such as to the dataset A text classification system may be built up from the following components:

Text representation: to convert documents into a set of features that can be handled by classification algorithms.

Feature reduction: to reduce the number of features to make classification algorithms efficient which can be implemented by methods including dimension reduction

Classifier training: to build up an autonomous classifier by using supervised learning algorithms.

Prediction: to generate labels or classifications for new documents by using the trained classifier.

Description of Solution

 

In order to provide a visualisation of the proposed system, a 3-layer model diagram has been developed and is shown in the diagram below.

Figure 8 3-tier architecture model of application

 

The project will consist of the web application, which is made up of the Django server, an S.Q.L. lite database for persisting user data for user authentication, forum data, poll data, and storage for the main document classification system. The system will be hosted on an A.W.S. infrastructure with an E.C2, and S3 bucket communication between layers will be handled by HTTP/IP protocol stacks.

The classification of documents will complete the following steps:

       After login, the user will upload documents for classification.

       The system will utilise optical character recognition for capturing data from documents where necessary.

       The data will be cleaned and prepared to increase the quality of results.

       Data will be split into training and test data.

       Machine learning algorithms will be applied to the training model.

       The resulting test model will be tested with the test data.

       The final user model will then be used for classification.

       The completed results will be displayed to the user.

5 Use Cases of NLP in Business

 

NLP tools are helping companies understand how their customers perceive them across all channels of communication, whether emails, product reviews, social media posts, surveys, and more.

 

Not only can AI tools be used to understand online conversations and how customers are talking about businesses, they can also be used to automate repetitive and time-consuming tasks, increase efficiency, and enable workers to focus on more fulfilling tasks.

 

Here are some of the main applications of NLP in business:

Sentiment Analysis

 

Sentiment analysis identifies emotions in text and classifies opinions as positive, negative, or neutral. You can see how it works by pasting text into this free sentiment analysis tool.

 

By analysing social media posts, product reviews, or online surveys, companies can gain insight into how customers feel about brands or products. For example, you could analyse tweets mentioning your brand in real-time and detect comments from angry customers right away.

 

Maybe you want to send out a survey to find out how customers feel about your level of customer service. By analysing open-ended responses to NPS surveys, you can determine which aspects of your customer service receive positive or negative feedback.

Language Translation

 

Machine translation technology has seen great improvement over the past few years, with Facebooks translations achieving superhuman performance in 2019.

 

Translation tools enable businesses to communicate in different languages, helping them improve their global communication or break into new markets.

 

You can also train translation tools to understand specific terminology in any given industry, like finance or medicine. So you dont have to worry about inaccurate translations that are common with generic translation tools.

Text Extraction

 

Text extraction enables you to pull out pre-defined information from text. If you deal with large amounts of data, this tool helps you recognize and extract relevant keywords and features (like product codes, colors, and specs), and named entities (like names of people, locations, company names, emails, etc).

 

Companies can use text extraction to automatically find key terms in legal documents, identify the main words mentioned in customer support tickets, or pull out product specifications from a paragraph of text, among many other applications. Sounds interesting? Heres a keyword extraction tool you can try.

Chatbots

 

Chatbots are AI systems designed to interact with humans through text or speech.

 

The use of chatbots for customer care is on the rise, due to their ability to offer 24/7 assistance (speeding up response times), handle multiple queries simultaneously, and free up human agents from answering repetitive questions.

 

Chatbots actively learn from each interaction and get better at understanding user intent, so you can rely on them to perform repetitive and simple tasks. If they come across a customer query theyre not able to respond to, theyll pass it onto a human agent.

Topic Classification

 

Topic classification helps you organize unstructured text into categories. For companies, its a great way of gaining insights from customer feedback.

 

Imagine youd like to analyze hundreds of open-ended responses to NPS surveys. How many responses mention your customer support? What percentage of customers talk about Pricing? With this topic classifier for NPS feedback, youll have all your data tagged in seconds.

 

Also, you can use topic classification to automate the process of tagging incoming support tickets and automatically route them to the right person.

Testing

The project will attempt to follow a test-driven development approach that will hopefully help to mitigate the project risks. This is a programming practice that instructs developers to write new code only if an automated test has failed. This avoids duplication of code. Test-Driven Development starts with designing and developing tests for every small functionality of an application. In the TDD approach, first, the test is developed, which specifies and validates what the code will do.

 

Figure 34-T.D.D. cycle

The T.D.D. cycle is

  1. Write a test case for some functionality

2.       Write the code that implements a solution to the test case.

  1. If the test case fails, change the code to make it successful, i.e., Refactoring.
  2. Repeat process for all test cases.

The primary goal of TDD is to make the code clearer, simple, and bug-free.

The application development will also make use of Ad hoc testing. This is the most common method of testing any software. Ad hoc boils down to the developer testing the code by running it after developing it, and verifying whether the intended behaviour occurs. Ad Hoc is manual testing; therefore it can be more time consuming then automatic modes of testing. That being said, it is the easiest to understand and only requires a knowledge of the system to perform. It is very similar to white box testing [21] as it is carried out by the developer.

During the development cycle, ad hoc test is being applied to this project as well as the use of automated tests is that the testing work is done for you by the system. You create a set of tests once, and then as you make changes to your app, you can check that your code still works as you originally intended, without having to perform time-consuming manual testing

The need for quality assurance within the project is an important aspect of the project, which is why I will be following the test automation pyramid devised by Mike Cohn (Cohn, 2010).

Figure 35-Test pyramid

The pyramid is a plan for the number of test cases that need to be created for each part of the system. I.e., most of the testing should be done at the functional layer rather than at the user interface layer.

According to (Cohn, 2010, p. 307), the importance of integrating testing into the process

rather than leaving it as something to be done later is of vital importance and should be followed from the start of the product development.

The Django framework provides a built-in tests class that can be instantiated with any functional or integration test cases for examples of these please see the appendix

2.       Create a list of tasks that will address these goals.

3.       Get the people who will be using the completed project to take the test.

Issues and Risks

 

Sourcing Data

This project would be deemed impossible without access to the appropriate data. There is a vast array of open-source data sets that have been made available by various organisations and individuals. One of the biggest factors associated with this kind of project is the need to provide a dataset that is capable of both training the classifier and is suitable for testing the classifier's algorithms. For this reason and to mitigate project risk I will be seeking a dataset in which the data has been pre-processed as much as possible.

 

Programming Experience in the field of Data Mining

 

The system requires a moderate amount of programming with several languages. Knowledge of the language Python is possessed; however, the use of several new modules specific to the task of data mining and machine learning will need to be learned. The execution of complex algorithms upon data sets will present a steep learning curve and could slow project progression. I will also have to learn the Django framework for the implementation of the proof of concept.

 

Quantity of Work

The project scope is vast, including the finding of a suitable dataset, implementation of a predictive model, and deployment in a three-tiered web cloud environment. To ensure that the project is complete on time, it is important to plan the development to a high slandered and endeavour to stick to the timeline as much as possible. If it gets to the point where it is deemed too great a task, the scope of the project will have to be reduced on the side of the web deployment, with the focus changing to the creation of the predictive model.
